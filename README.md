# Korea-Japan Trip
釜山と福岡の観光地を地図上で直感的に確認できるWebサービスです。
「ハッカソン大会において、釜山と福岡を訪れる旅行者のために役立つ情報プラットフォームを構築したい」という思いからチームで企画・開発しました。

## プロジェクト概要
- **開発期間:** 2025年11月 (約2週間)
- **開発人数:** 3名 (ハッカソンチーム開発・準優勝)
- **デプロイURL:** https://korea-japan-trip.onrender.com

## 使用技術
- **Frontend:** HTML, CSS, JavaScript (モバイル対応レスポンシブデザイン)
- **Library:** Leaflet.js (インタラクティブマップ描画)
- **Database:** Firestore
- **API:** Open-Meteo API (天気情報), ExchangeRate-API (為替レート), 翻訳API

## 主な機能
1. **直感的な地図表示とフィルタリング:** Leaflet.jsを使用し、観光・グルメ・交通などのカテゴリ別にピンを絞り込めるマップを実装しました。
2. **リアルタイムな旅行情報:** 外部APIと連携し、選択した地域のリアルタイムな「天気」や、日韓の「現在の為替レート」を自動計算して表示します。
3. **ユーザー参加型のレビュー機能:** Firestoreを利用し、各観光地に対する「いいね」や「星評価付きレビュー」をリアルタイムで投稿・反映できます。
4. **多言語対応と翻訳機能:** 韓国語・日本語のUI切り替えに加え、ユーザーが残したレビューをボタン一つで双方向に翻訳できる機能を実装し、両国のユーザーがスムーズに情報共有できるようにしました。

## 工夫した点・トラブルシューティング
本プロジェクトでは、限られた期間内でのチーム開発において直面した課題を以下のように解決し、完成度を高めました。

**1. 納期を守るための柔軟な「技術的ピボット（方針転換）」**
当初は未経験の技術（Next.js、Supabase等）での開発に挑戦しましたが、2週間という短い期間の中で学習コストがかさみ、スケジュールが厳しくなる課題に直面しました。そこでチームに状況を共有し、確実に扱える技術（Vanilla JS、Firebase）へ素早く切り替える決断を下しました。理想の技術への執着よりも「現実的な解決策を見つけてサービスを完成させる実行力」を最優先しました。

**2. コミュニケーションを通じた精神的成長と目標達成**
初めてのチーム開発で、最初は自分の意見をうまく伝えられずプレッシャーを感じていました。しかし、チームに迷惑をかけたくないという思いからDiscord等で積極的にコミュニケーションを取り、徹夜で開発をやり遂げました。結果として大会で2位を受賞し、困難な状況でもチームと協力して課題を乗り越える喜びと打たれ強さを学ぶことができました。
